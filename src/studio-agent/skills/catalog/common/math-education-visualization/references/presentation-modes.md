# 呈现模式

## 一、静态呈现

静态呈现用于展示一个稳定状态，或把多个稳定状态并列出来给学生比较。

常见形式：

- 单张图
- 多图并列
- 一个页面内的多子图
- 主图加局部放大图

适合：

- 看结构
- 看分类讨论
- 看不同情况对比
- 看几何对象的多视角关系
- 看一个复杂对象被拆成几个部分

常用手法：

### 1. 分解呈现

把复杂对象拆开，不要让学生一次吞完整体。

例如：

- 三维对象拆成几个二维投影
- 分段函数拆成不同定义域下的多个子图
- 一个复杂构造拆成几个局部区域

### 2. 视角呈现

同一对象用不同观察视角共同呈现。

例如：

- 主视图、侧视图、俯视图并列
- 主图展示全局，旁边放局部放大

风险：

- 信息同时出现太多，学生不知道先看哪

默认应对：

- 用并列子图代替超载单图
- 通过排版把比较关系做清楚

## 二、伪动态呈现

伪动态呈现本质上还是静态媒介，但它把时间过程摊平到空间里，让学生按步骤看。

常见形式：

- 分步图
- 多关键帧并列
- 推导过程的层层展开
- 单图中的轨迹、残影、历史路径

适合：

- 看步骤
- 看构造过程
- 看代数推导
- 看算法或迭代的阶段变化
- 看连续变化中的几个关键节点

常用手法：

### 1. 多时间帧分镜

把一个过程抽成几个关键时刻排开。

例如：

- `t1 -> t2 -> t3`
- “原状态 -> 中间状态 -> 最终状态”

### 2. 趋势与轨迹暗示

在一张图内用残影、轨迹线、箭头说明对象从哪里来、往哪里去。

### 3. 逻辑过程呈现

用于公式推导、证明步骤、算法更新，把逻辑前后关系变成版面顺序。

风险：

- 学生不知道阅读顺序
- 每一步跨度太大，无法和前一步对上

默认应对：

- 明确排阅读路径
- 每一步只推进少量信息
- 相邻步骤保持足够的结构连续性

## 三、动态呈现

动态呈现有真实时间轴，适合“连续变化本身就是内容”的情况。

包括：

- 动画
- 参数连续变化
- 滑块驱动的交互可视化

适合：

- 参数变化
- 连续运动
- 极限逼近
- 图形变换
- 从一个数学对象过渡到另一个对象

动态内部再分两类：

### 1. 量变

对象的定义没有变，只是参数、位置、精度等在连续调整。

例如：

- 二次函数开口随参数变化
- 向量连续旋转
- 逼近曲线越来越贴近目标曲线

### 2. 质变

对象发生了概念层级上的跳跃，不只是“变一点”，而是“变成另一种数学状态”。

例如：

- 多边形趋向圆
- 离散和走向积分解释
- 公式拆解后重组为几何图形

风险：

- 动画太热闹，学生只看到运动，没看到数学

默认应对：

- 先定义哪一部分必须保持身份连续
- 限制同时运动的对象数量
- 不要让镜头运动和对象运动同时过于复杂

## 快速决策表

- 如果要“停下来观察和比较”，选静态
- 如果要“按步骤理解过程”，选伪动态
- 如果要“理解连续变化本身”，选动态

如果不确定，优先顺序通常是：

`静态 > 伪动态 > 动态`

原因不是静态更高级，而是它对理解的控制更稳，出错成本更低。
