# 色彩设计规范

## 总原则

颜色的任务是帮助理解，不是制造“设计感”。

任何颜色的引入都应当有明确理由。默认先做减法，再做配色。

## 先分层，再配色

画面至少分成两层：

- 背景层
- 研究对象层

背景层通常包括：

- 坐标轴
- 辅助框架
- 当前不是重点的几何对象
- 普通说明文字

默认做法：

- 背景层使用中性色
- 研究对象层才使用语义色彩

如果背景对象突然变成当前研究重点，它才能被提升出背景层。

## 三种配色场景

### 一、展示场景

用于：

- 常规讲解
- 定义说明
- 长时间阅读
- 稳定的符号和图形对应

设计目标：

- 区分对象
- 保持平衡
- 不制造刺激性干扰

默认规则：

- 低饱和度
- 同层对象明度一致
- 同一语义保持同色对应

白场默认起点：

- 背景：`hsl(210, 15%, 98%)`
- 对象色统一参数：饱和度 `25%`，明度 `45%`

### 二、突出场景

用于：

- 强调关键点
- 指出错误
- 引发顿悟
- 瞬间拉住注意力

设计目标：

- 打断平稳阅读
- 强制聚焦重点

默认规则：

- 提高饱和度
- 控制明度，不让颜色发白发虚
- 只突出少量关键对象，不能全画面一起高亮

白场默认起点：

- 饱和度 `90%`
- 明度 `45%`

黑场默认起点：

- 饱和度 `85%`
- 明度 `60%`

### 三、动态演变场景

用于：

- 极限过程
- 逼近过程
- 参数变化
- 对象状态演化

这里最重要的不是“颜色变了”，而是“颜色为什么这样变”。

动态演变分两类：

#### 1. 质变

对象的数学身份发生变化。

例如：

- 离散求和走向积分
- 多边形趋向圆
- 代数式重组成几何对象

默认颜色策略：

- 允许色相跨越
- 让学生感到“这是另一个状态”

#### 2. 量变

对象身份不变，只是在精度、强度、成熟度上变化。

例如：

- 逼近越来越准
- 迭代越来越稳定
- 项数增加后展开更完整

默认颜色策略：

- 保持色相不变
- 调整饱和度和透明度

一句话记忆：

- 质变靠换色相
- 量变靠同色渐强

不要把这两套逻辑混用，否则学生不知道自己看到的究竟是“换了对象”还是“同一对象变强了”。

## 动态插值起点

白场背景下可用：

- 明度锚点：`45%`
- 饱和度：`20% -> 90%`
- 透明度：`0.1 -> 1.0`

黑场背景下可用：

- 明度锚点：`60%`
- 饱和度：`30% -> 85%`
- 透明度：`0.1 -> 0.8`

## 直接可用的默认判断

如果你只想快速做决定，用这套：

1. 普通讲解：展示场景
2. 要强调某一步：突出场景
3. 要表现连续变化：动态演变场景

然后再问一句：

- 这是同一对象在变强，还是对象本质变了？

这个问题会直接决定你该不该改色相。
