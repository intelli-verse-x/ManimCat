# 图形元素与视觉变量

## 三层对象

任何数学可视化都至少涉及三层：

### 1. 数学对象

抽象层的对象，比如：

- 函数
- 向量
- 点集
- 极限过程
- 概率大小

### 2. 图形元素

你用什么把数学对象画出来，比如：

- 点
- 线
- 面
- 区域填充
- 标签
- 轨迹

### 3. 视觉变量

图形元素的属性，比如：

- 色相
- 明度
- 饱和度
- 尺寸
- 透明度
- 形状
- 线型

正确顺序不是“先想颜色”，而是：

数学对象 -> 是否需要图形元素 -> 用哪种图形元素 -> 用什么视觉变量

## 降噪原则

先问“需不需要画”，再问“画成什么样”。

默认准则：

- 每一个可见元素都应当承载数学逻辑
- 不能承载逻辑的元素优先删除
- 可以退到背景层的元素不要留在前景

典型降噪动作：

- 减少图内文字
- 去掉不必要网格线
- 弱化坐标轴箭头和刻度
- 不重复标注已经显然成立的关系
- 当主体变化后，及时把旧主体降成背景

例子：

- 研究导数定义时，函数曲线可能只是背景，真正主体是切线和增量三角形
- 研究极限逼近时，原函数可以弱化，逐步逼近的对象要成为主体

## 视觉变量的四种感知属性

### 1. 选择性

让学生一眼从一堆对象里挑出同类。

适合的变量：

- 色相

谨慎使用：

- 尺寸

不适合作为首选：

- 形状，尤其在对象很多时

结论：

如果目标是“分类”，默认先想颜色，不要先想形状。

### 2. 结合性

改变变量后，群体是否还能保持“平等感”。

适合保持平等的变量：

- 色相
- 形状
- 线型

容易破坏平等感的变量：

- 尺寸
- 明度

结论：

如果两个对象在数学地位上平等，不要一个画得更大、更黑、更亮。

### 3. 顺序性

让学生直觉感到“有先后、有强弱、有高低”。

适合的变量：

- 明度
- 透明度
- 尺寸

不适合：

- 色相
- 形状

结论：

如果目标是表达阶段递进，优先用浅到深、弱到强、小到大。

### 4. 定量性

让学生能粗略感到比例与倍数。

适合的变量：

- 位置
- 长度
- 面积
- 尺寸

不适合：

- 色相
- 明度

结论：

如果目标是表达“谁是两倍”“谁更长”“谁更大”，靠位置和长度，不靠颜色深浅。

## 三类信息的默认映射

### 一、定性信息

例子：

- 不同函数簇
- 不同点集
- 实线与渐近线
- 两类解法

目标：

- 分类
- 区分
- 同时保持平等

推荐：

- 色相
- 形状
- 线型

默认不优先：

- 尺寸

### 二、顺序信息

例子：

- 迭代阶段
- 极限逼近过程
- 概率由低到高
- 证明步骤先后

目标：

- 展现递进
- 展现阶段感

推荐：

- 明度
- 透明度
- 尺寸

默认不优先：

- 色相
- 形状

### 三、定量信息

例子：

- 向量模长
- 导数斜率大小
- 区域面积对比
- 函数值的大小差异

目标：

- 表达比例
- 表达数值大小
- 表达倍数关系

推荐：

- 位置
- 长度
- 面积
- 尺寸

默认不优先：

- 色相
- 明度

## 作图前的简短检查

作图前快速问自己：

1. 我现在表达的是分类、顺序还是数量？
2. 我有没有把不该抢眼的对象画得太重？
3. 我是不是用颜色做了本该由位置或长度承担的工作？
4. 有没有元素删掉以后反而更清楚？
